# AI Study Assistant — Design Spec

**Date:** 2026-03-21
**Status:** Approved

---

## Overview

Loci is a study tool. This spec adds AI-powered study features that transform the EPUB reader into a complete revision environment. The design is built around three study modes — active recall, deep understanding, and synthesis — delivered through a single, beautiful AI chat panel that feels like a personal tutor rather than a feature dashboard.

---

## Goals

1. Let students capture thoughts while reading without friction (no highlight required)
2. Generate, save, and review flashcards from chapter content
3. Provide a persistent, context-aware AI tutor that knows the book, notes, and flashcards
4. Surface study progress in the book detail modal
5. Keep the reader visually calm — AI is additive, never intrusive

---

## What We're Building

### 1. Freeform Note-Taking (two types)

**A. Chapter notes** — freeform note tied to the current chapter. No text selection required. Student taps a button, gets a blank editor, types freely. Filed under the chapter in the sidebar Notes tab alongside existing highlight-anchored annotations.

**B. Book scratchpad** — one persistent free-form space per book. Lives alongside the reader (toggle). Used for synthesis: connecting ideas across chapters, exam summaries, revision lists. Editable only inside the reader; shown as a read-only preview in the book detail modal.

### 2. Flashcards

- Generated by the AI on request ("Make flashcards" chip or direct chat message)
- Each card has a **front** (concept/quote/term) and **back** (explanation/context)
- Cards are created inline in the chat thread as flippable card components, and auto-saved to Supabase
- Grouped by chapter in the sidebar and modal
- Reviewable from both the reader (AI panel) and the book detail modal
- Exportable as plain text for use in other revision tools

### 3. AI Study Assistant Panel

A right-side chat panel toggled by the ✦ sparkle button in the reader header. The panel is chat-first — one continuous conversation — with context-aware quick-action chips above the input.

**Context the AI always has:**
- Current chapter text (truncated to ~40k chars if needed)
- All chapter notes for the current chapter
- The book scratchpad (full content)
- All highlight-anchored annotations for the book
- All saved flashcards for the book (front + back text)
- Currently selected text in the epub (if any)

**Chip logic — context-aware, max 3 visible at once:**

| Context state | Chips shown |
|---|---|
| Reading (default) | Summarise chapter · Quiz me · Make flashcards |
| Text selected in epub | Explain this · Save as flashcard |
| Chapter has notes | Review my notes · Quiz from notes |
| Has saved flashcards | Review flashcards · Test me |

Chips are smart shortcuts — tapping one sends a pre-formed message to the AI. They do not replace the open input field.

**AI identity:**
- Icon: ✦ (warm gold, `--accent-warm`)
- User messages: dark bubble (`#1A1917`), right-aligned
- AI messages: warm paper bubble (`#F8F7F4`), left-aligned with ✦ avatar
- Flashcard cards: rendered inline in the thread as flippable front/back cards, auto-saved on generation
- Context pill at top of panel: shows current chapter title

### 4. Book Detail Modal — Study Data

The modal gains a stats card in the left panel and two new sections in the right scroll area:

**Left panel stats card** (below the Read button):
- Reading time (existing)
- Flashcards (count, warm gold)
- Chapter notes (count, warm gold)
- Highlights (count)

**Flashcards section** (right panel, above Notes):
- Grouped by chapter range (e.g. "Chapter I–III")
- Shows card count + last reviewed date per group
- "Review" button per group — launches flashcard review session within the AI panel
- "↗ Export" button — downloads all flashcards as plain text

**Scratchpad preview** (right panel):
- 3-line truncated read-only preview of the scratchpad
- "Open in reader →" link — opens the book and activates the scratchpad panel
- If no scratchpad content: hidden

**Notes** renamed to **"Highlights & notes"** to reflect that both highlight-anchored annotations and freeform chapter notes appear here.

## Architecture

### AI Backend

**Supabase Edge Function:** `supabase/functions/ai-study/index.ts`

Handles all AI requests. Authenticated via Clerk JWT (same pattern as all other Supabase calls). Provider is configured server-side via environment variables — no API keys are ever exposed to the client.

```
POST /functions/v1/ai-study
Authorization: Bearer <clerk-jwt>
{
  "action": "chat" | "summarise" | "quiz" | "flashcards" | "explain",
  "messages": [{ "role": "user" | "assistant", "content": string }],
  "context": {
    "chapterText": string,
    "chapterNotes": string[],
    "scratchpad": string,
    "annotations": { quote: string, note: string }[],
    "flashcards": { front: string, back: string }[],
    "selectedText": string | null
  }
}
```

Returns: streaming text via `text/event-stream` (SSE).

**Provider abstraction:**

```typescript
// Env vars on Supabase Edge Function
AI_PROVIDER = "anthropic" | "google"
ANTHROPIC_API_KEY = "sk-ant-..."
GOOGLE_AI_API_KEY = "AIza..."
```

The function reads `AI_PROVIDER` at runtime and routes to the appropriate SDK. Both providers receive the same structured prompt. Default provider: `anthropic`. Can be switched without code changes.

**Models:**
- Anthropic: `claude-haiku-4-5-20251001` (fast, low cost)
- Google: `gemini-2.0-flash` (fast, low cost)

**System prompt (study tutor persona):**

```
You are a study assistant helping a student understand and revise their reading material.
You have access to the current chapter text, the student's notes, their scratchpad,
their saved highlights, and their flashcards.

Be concise, clear, and pedagogically sound. When generating flashcards, output them
as a JSON array: [{"front": "...", "back": "..."}].
When quizzing, ask one question at a time and wait for the student's answer before
revealing the correct answer. Never make up information not present in the provided text.
```

### Client-Side Service

**`src/services/aiStudyService.ts`**

```typescript
interface StudyContext {
  chapterText: string
  chapterNotes: string[]
  scratchpad: string
  annotations: { quote: string; note: string }[]
  flashcards: { front: string; back: string }[]
  selectedText: string | null
}

interface Message {
  role: 'user' | 'assistant'
  content: string
}

// Returns an async generator that yields text chunks (streaming)
async function* sendMessage(
  supabase: SupabaseClient,
  action: string,
  messages: Message[],
  context: StudyContext
): AsyncGenerator<string>
```

Streams the response using `fetch` + `ReadableStream`. Yields text chunks to the component as they arrive.

## Data Model

### New table: `flashcards`

```sql
create table public.flashcards (
  id uuid primary key default gen_random_uuid(),
  user_id text not null,
  book_id uuid not null references public.books(id) on delete cascade,
  chapter_href text not null,
  front text not null,
  back text not null,
  created_at timestamptz default now(),
  last_reviewed_at timestamptz,
  review_count integer not null default 0
);

alter table public.flashcards enable row level security;
create policy "Users manage own flashcards"
  on public.flashcards for all
  using ((auth.jwt() ->> 'sub') = user_id);
```

### Extended: `annotations` table

Add a `type` discriminator to support chapter notes alongside existing highlight annotations:

```sql
alter table public.annotations
  add column type text not null default 'highlight';
-- type: 'highlight' (existing, quote required) | 'chapter_note' (new, empty quote)
```

Chapter notes use the existing `annotations` row shape:
- `quote`: `''` (empty — no highlight)
- `note`: the student's freeform text
- `href`: current chapter spine href
- `type`: `'chapter_note'`

This keeps the notes export unified and the sidebar Notes tab one data source.

### New table: `scratchpad`

```sql
create table public.scratchpad (
  user_id text not null,
  book_id uuid not null references public.books(id) on delete cascade,
  content text not null default '',
  updated_at timestamptz default now(),
  primary key (user_id, book_id)
);

alter table public.scratchpad enable row level security;
create policy "Users manage own scratchpad"
  on public.scratchpad for all
  using ((auth.jwt() ->> 'sub') = user_id);
```

## New Components

### `src/components/StudyPanel.tsx`

Right-side chat panel. Props:

```typescript
interface StudyPanelProps {
  isOpen: boolean
  onClose: () => void
  context: StudyContext
  supabase: SupabaseClient
  onFlashcardsGenerated: (cards: { front: string; back: string }[], chapterHref: string) => void
}
```

**Internal state:**
- `messages: Message[]` — chat history
- `input: string` — current input value
- `streaming: boolean` — whether AI is currently responding
- `activeChips: string[]` — computed from context, max 3

**Chip computation logic:**
```typescript
function computeChips(context: StudyContext, hasFlashcards: boolean, hasChapterNotes: boolean): string[] {
  if (context.selectedText) return ['Explain this', 'Save as flashcard']
  if (hasChapterNotes && hasFlashcards) return ['Review my notes', 'Quiz me', 'Review flashcards']
  if (hasChapterNotes) return ['Review my notes', 'Summarise chapter', 'Quiz from notes']
  if (hasFlashcards) return ['Summarise chapter', 'Quiz me', 'Review flashcards']
  return ['Summarise chapter', 'Quiz me', 'Make flashcards']
}
```

**Flashcard parsing:** When the AI returns a message containing a JSON array of `{front, back}` objects, the component renders them as `FlashCard` components inline in the thread and calls `onFlashcardsGenerated`.

### `src/components/FlashCard.tsx`

A flippable card component used both inside `StudyPanel` (inline in chat) and in the flashcard review session.

```typescript
interface FlashCardProps {
  front: string
  back: string
  isReview?: boolean  // true = shows nav buttons (prev/next/mark known)
}
```

CSS 3D flip animation (rotateY). Front: warm gradient background, Lora italic. Back: dark background (`#1A1917`), DM Sans, white text.

### `src/components/Scratchpad.tsx`

Right-side panel (same overlay slot as StudyPanel, mutually exclusive). Props:

```typescript
interface ScratchpadProps {
  isOpen: boolean
  onClose: () => void
  bookId: string
  supabase: SupabaseClient
}
```

- Loads scratchpad content on mount from Supabase
- Auto-saves on blur with 1s debounce
- Plain textarea, no formatting — intentionally minimal
- DM Sans 14px, warm background, generous padding

### `src/services/flashcardService.ts`

```typescript
getFlashcards(supabase, bookId): Promise<Flashcard[]>
saveFlashcards(supabase, userId, bookId, chapterHref, cards): Promise<Flashcard[]>
deleteFlashcard(supabase, id): Promise<void>

```

### `src/services/scratchpadService.ts`

```typescript
getScratchpad(supabase, bookId): Promise<string>
saveScratchpad(supabase, userId, bookId, content): Promise<void>
```

### `src/hooks/useFlashcards.ts`

Mirrors the `useAnnotations` / `useBookmarks` pattern. Manages local state + async Supabase sync. Exposed from `Reader.tsx` and passed down to `StudyPanel` and `BookDetailModal`.

### `src/hooks/useScratchpad.ts`

Loads and saves the scratchpad. Exposes `content: string`, `setContent: (s: string) => void`, `saving: boolean`.

## Reader Changes

### Header

Add ✦ sparkle button to the header right section, between the Notes button and ThemeToggle:

```
[🔍 Search] [🔖 Bookmark] [✎ Notes] [✦ Study] [☀️/🌙 Theme]
```

The ✦ button is styled distinctively — dark background when active (panel open), warm gold icon.

### Right-side panel slot

The reader already has a right-side notes pane slot. The scratchpad and study panel share this slot — only one can be open at a time. The Notes button opens the notes pane (existing), the Scratchpad button (new, in AudioBar settings) opens the scratchpad, the ✦ button opens the study panel.

### Chapter notes integration

The existing `SelectionBubble` only appears on text selection. We add a "+ Note" button to the reader, visible as a small floating action button anchored to the bottom of the epub viewer (not in the header — keeps the header clean). Tapping it opens a blank note editor (`type: 'chapter_note'`) without requiring a selection.

The annotation sidebar Notes tab already renders all annotations; chapter notes render the same way but without a quote block.

### Context building

`Reader.tsx` assembles the `StudyContext` object and passes it to `StudyPanel`:

```typescript
const studyContext: StudyContext = {
  chapterText: epub.getCurrentText(),
  chapterNotes: annotations
    .filter(a => a.href === currentHref && a.type === 'chapter_note')
    .map(a => a.note),
  scratchpad: scratchpad.content,
  annotations: annotations
    .filter(a => a.type === 'highlight' && a.note)
    .map(a => ({ quote: a.quote, note: a.note })),
  flashcards: flashcards.map(f => ({ front: f.front, back: f.back })),
  selectedText: currentSelection ?? null,
}
```

## Sidebar Changes

The Sidebar Notes tab renders a unified list of all annotation types:

- `type === 'highlight'` → existing AnnotationCard with quote block (italic Lora quote + note text)
- `type === 'chapter_note'` → NoteCard without quote block (just note text + chapter label + date)

No new tabs. The Notes tab becomes slightly richer but stays one surface.

## Book Detail Modal Changes

See "Book Detail Modal — Study Data" section above. Implementation:

- On mount: load flashcards via `getFlashcards(supabase, bookId)` alongside existing annotation load
- Load scratchpad preview via `getScratchpad(supabase, bookId)` (first 200 chars)
- Group flashcards by `chapter_href` for display
- "Review" button in modal: opens the book, passes a flag to open the StudyPanel in review mode for that chapter group
