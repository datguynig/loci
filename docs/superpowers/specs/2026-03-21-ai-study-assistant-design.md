# AI Study Assistant — Design Spec

**Date:** 2026-03-21
**Status:** Approved (revised after two spec reviews)

---

## Overview

Loci is a study tool. This spec adds AI-powered study features that transform the EPUB reader into a complete revision environment. The design is built around three study modes — active recall, understanding, and synthesis — delivered through a single, beautiful AI chat panel that feels like a personal tutor rather than a feature dashboard.

---

## Goals

1. Let students capture thoughts while reading without friction (no highlight required)
2. Generate, save, and review flashcards from chapter content
3. Provide a persistent, context-aware AI tutor that knows the book, notes, and flashcards
4. Surface study progress in the book detail modal
5. Keep the reader visually calm — AI is additive, never intrusive

---

## What We're Building

### 1. Freeform Note-Taking (two types)

**A. Chapter notes** — freeform note tied to the current chapter. No text selection required. A `+ Note` floating action button sits anchored to the bottom-right corner of the epub viewer (above the audio bar, not in the header). Tapping it opens a blank note editor. Filed under the chapter in the sidebar Notes tab alongside existing highlight annotations.

**B. Book scratchpad** — one persistent free-form space per book. Toggled by a notebook icon button in the reader header. Lives as a right-side overlay panel (same slot as the Study panel — mutually exclusive). Used for synthesis: connecting ideas across chapters, exam summaries, revision lists. Editable only inside the reader; shown as a read-only 3-line preview in the book detail modal.

### 2. Flashcards

- Generated by the AI on request ("Make flashcards" chip or direct chat message)
- Each card has a **front** (concept/quote/term) and **back** (explanation/context)
- Cards drop inline into the chat thread as flippable `FlashCard` components and are auto-saved to Supabase
- Grouped by chapter in the sidebar Study tab and in the book detail modal
- Reviewable from both the reader (Study panel) and the book detail modal
- Exportable as plain text

### 3. AI Study Assistant Panel

A right-side chat panel (320px, same width as the existing notes pane) toggled by the ✦ sparkle button in the reader header. The panel is chat-first — one continuous conversation — with context-aware quick-action chips above the input.

**Context the AI always has:**
- Full current chapter text (see note on extraction below)
- All chapter notes for the current chapter
- The book scratchpad (full content)
- All highlight-anchored annotations for the book (quote + note)
- All saved flashcards for the book (front + back)
- Currently selected text in the epub (if any)

**Full chapter text extraction:** `epub.getCurrentText()` returns the text of the currently rendered view, which in paginated/spread mode may be only one page. For AI context, a separate `getFullChapterText(): string` method is added to `useEpub` that extracts text from the chapter's full rendered document (not just the viewport), regardless of layout mode.

**Chip logic — context-aware, max 3 visible at once:**

| Context state | Chips shown |
|---|---|
| Reading (default) | Summarise chapter · Quiz me · Make flashcards |
| Text selected in epub | Explain this · Save as flashcard |
| Chapter has notes + flashcards | Review my notes · Quiz me · Review flashcards |
| Chapter has notes only | Review my notes · Summarise chapter · Quiz from notes |
| Has flashcards only | Summarise chapter · Quiz me · Review flashcards |

Chips are smart shortcuts — tapping one sends a pre-formed message into the chat. The open input field is always available.

**AI identity:**
- Icon: ✦ (warm gold, `--accent-warm`)
- User messages: dark bubble (`#1A1917`), right-aligned, DM Sans 13px
- AI messages: warm paper bubble (`#F8F7F4`), left-aligned, Lora 13px, ✦ avatar
- Flashcard cards: rendered inline in the thread, saved automatically on generation
- Context pill at top of panel: shows current chapter title — updates on chapter change

### 4. Book Detail Modal — Study Data

**Left panel stats card** (replaces the standalone "Time spent reading" text):

```
Reading time   |  4h 23m
Flashcards     |  24        ← warm gold
Chapter notes  |   6        ← warm gold
Highlights     |  18
```

**Flashcards section** (right panel, above Notes):
- Grouped by chapter href, displayed as chapter label
- Shows card count + last reviewed date per group
- "Review" button per group — calls `onStudy({ chapterHref })` (see Reader Changes)
- "↗ Export" — downloads all flashcards as plain text (front / back pairs)

**Scratchpad preview** (right panel, below Flashcards):
- 3-line truncated read-only preview
- "Open in reader →" — calls `onStudy({ panel: 'scratchpad' })`
- Hidden if scratchpad content is empty

**Notes renamed** to **"Highlights & notes"** — covers both `type: 'note'` (highlight-anchored) and `type: 'chapter_note'` (freeform) annotations.

## Architecture

### AI Backend

**Supabase Edge Function:** `supabase/functions/ai-study/index.ts`

Authenticated via Clerk JWT. Provider configured server-side — no API keys exposed to the client.

```
POST /functions/v1/ai-study
Authorization: Bearer <clerk-jwt>
Content-Type: application/json

{
  "action": "chat" | "summarise" | "quiz" | "flashcards" | "explain",
  "messages": [{ "role": "user" | "assistant", "content": string }],
  "context": {
    "chapterText": string,
    "chapterNotes": string[],
    "scratchpad": string,
    "annotations": { "quote": string, "note": string }[],
    "flashcards": { "front": string, "back": string }[],
    "selectedText": string | null
  }
}
```

Returns: `text/event-stream` (SSE), one `data: <chunk>` per token.

**Provider abstraction — env vars:**

```
AI_PROVIDER=anthropic   # or "google"
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_AI_API_KEY=AIza...
```

The function branches on `AI_PROVIDER` at runtime. Both providers receive the same system prompt and messages. Switching provider requires only an env var change — no code changes.

**Models:**
- Anthropic: `claude-haiku-4-5-20251001`
- Google: `gemini-2.0-flash`

**System prompt (study tutor persona):**

```
You are a study assistant helping a student understand and revise their reading material.
You have access to the current chapter text, the student's notes, their scratchpad,
their saved highlights, and their flashcards.

Be concise, clear, and pedagogically sound. When the user asks for flashcards, output
them as a JSON array on a single line: [{"front":"...","back":"..."},...].
When quizzing, ask one question at a time and wait for the answer before revealing
the correct response. Never fabricate information not present in the provided text.
```

### Client-Side AI Service

**`src/services/aiStudyService.ts`**

```typescript
export interface StudyContext {
  chapterText: string
  chapterNotes: string[]
  scratchpad: string
  annotations: { quote: string; note: string }[]
  flashcards: { front: string; back: string }[]
  selectedText: string | null
}

export interface Message {
  role: 'user' | 'assistant'
  content: string
}

// Returns an async generator yielding text chunks as they stream
export async function* sendStudyMessage(
  supabase: SupabaseClient,
  action: string,
  messages: Message[],
  context: StudyContext,
): AsyncGenerator<string>
```

Uses `fetch` with `ReadableStream` + `getReader()` to parse SSE chunks. Callers `for await` over chunks to build the streaming response.

## Data Model

### TypeScript types

**Updated `Annotation` interface** in `src/services/annotationService.ts`:

```typescript
export interface Annotation {
  id: string
  bookId: string
  href: string
  quote: string        // empty string for chapter_note type
  note: string
  type: 'note' | 'chapter_note'   // NEW field
  createdAt: number
}
```

All existing call sites that construct an `Annotation` object must be updated to pass `type: 'note'`. This affects: `useAnnotations.ts` (the `addAnnotation` helper), `Reader.tsx` (where annotations are constructed), and any test fixtures.

**New `Flashcard` interface** in `src/services/flashcardService.ts`:

```typescript
export interface Flashcard {
  id: string
  bookId: string
  chapterHref: string
  front: string
  back: string
  createdAt: number
  lastReviewedAt: number | null
  reviewCount: number
}
```

### DB migrations

**Migration 1 — `annotations` type column:**

```sql
-- Add type discriminator; existing rows get 'note' (matches what saveAnnotationToSupabase already writes)
alter table public.annotations
  add column if not exists type text not null default 'note';
```

Update `saveAnnotationToSupabase` to pass `annotation.type` instead of the hardcoded `'note'` string:

```typescript
// Before:
type: 'note',
// After:
type: annotation.type,
```

Update `getAnnotationsFromSupabase` to map `row.type` back into the returned object:

```typescript
return data.map((row) => ({
  id: row.id,
  bookId: row.book_id,
  href: row.spine_href,
  quote: row.quote,
  note: row.note ?? '',
  type: (row.type ?? 'note') as Annotation['type'],   // NEW
  createdAt: new Date(row.created_at).getTime(),
}))
```

**Migration 2 — `flashcards` table:**

```sql
create table public.flashcards (
  id uuid primary key default gen_random_uuid(),
  user_id text not null,
  book_id uuid not null references public.books(id) on delete cascade,
  chapter_href text not null,
  front text not null,
  back text not null,
  created_at timestamptz default now(),
  last_reviewed_at timestamptz,
  review_count integer not null default 0
);

alter table public.flashcards enable row level security;
create policy "Users manage own flashcards"
  on public.flashcards for all
  using ((auth.jwt() ->> 'sub') = user_id);
```

**Migration 3 — `scratchpad` table:**

```sql
create table public.scratchpad (
  user_id text not null,
  book_id uuid not null references public.books(id) on delete cascade,
  content text not null default '',
  updated_at timestamptz default now(),
  primary key (user_id, book_id)
);

alter table public.scratchpad enable row level security;
create policy "Users manage own scratchpad"
  on public.scratchpad for all
  using ((auth.jwt() ->> 'sub') = user_id);
```

## New Hooks

### `src/hooks/useFlashcards.ts`

Mirrors the `useBookmarks` pattern. Manages local `Flashcard[]` state + async Supabase sync.

```typescript
export function useFlashcards(supabase, userId, bookId) {
  return {
    flashcards: Flashcard[],
    addFlashcards: (chapterHref: string, cards: { front: string; back: string }[]) => Promise<void>,
    deleteFlashcard: (id: string) => Promise<void>,
    markReviewed: (id: string) => Promise<void>,
  }
}
```

### `useAnnotations` hook — updated `addAnnotation` signature

The existing `addAnnotation` helper in `useAnnotations.ts` constructs and saves an `Annotation` internally. It must accept an optional `type` parameter:

```typescript
// Current (inferred from usage):
addAnnotation: (href: string, quote: string, note: string) => void

// Updated:
addAnnotation: (href: string, quote: string, note: string, type: Annotation['type'] = 'note') => void
```

The `Annotation` object constructed inside the hook passes `type` through to both localStorage and `saveAnnotationToSupabase`.

The `+ Note` floating button in `Reader.tsx` calls:
```typescript
addAnnotation(currentHref, '', noteText, 'chapter_note')
```

The existing `SelectionBubble` save path calls (unchanged):
```typescript
addAnnotation(selectedHref, quote, note)   // type defaults to 'note'
```

There is no separate `addChapterNote` helper — the same `addAnnotation` function handles both types via the optional `type` argument.
```

### `src/hooks/useScratchpad.ts`

```typescript
export function useScratchpad(supabase, userId, bookId) {
  return {
    content: string,
    setContent: (s: string) => void,  // updates local state immediately
    saving: boolean,                  // true during debounced Supabase write
  }
}
```

Auto-saves to Supabase on content change with a 1-second debounce.

## New Components

### `src/components/StudyPanel.tsx`

```typescript
interface StudyPanelProps {
  isOpen: boolean
  onClose: () => void
  userId: string                  // needed for flashcard save
  context: StudyContext
  supabase: SupabaseClient
  onFlashcardsGenerated: (cards: { front: string; back: string }[], chapterHref: string) => void
  reviewMode?: { chapterHref: string }   // if set, open directly into flashcard review for this chapter
}
```

**Internal state:**
- `messages: Message[]` — full chat history for this session (cleared on panel close)
- `input: string`
- `streaming: boolean`

**Flashcard parsing:** After each complete AI message, scan the text for a JSON array matching `[{"front":...,"back":...}]`. If found: render each item as a `<FlashCard>` in the thread; call `onFlashcardsGenerated` so the parent saves them via `useFlashcards`.

**Chip computation:**

```typescript
function computeChips(context: StudyContext, hasChapterNotes: boolean, hasFlashcards: boolean): string[] {
  if (context.selectedText) return ['Explain this', 'Save as flashcard']
  if (hasChapterNotes && hasFlashcards) return ['Review my notes', 'Quiz me', 'Review flashcards']
  if (hasChapterNotes) return ['Review my notes', 'Summarise chapter', 'Quiz from notes']
  if (hasFlashcards) return ['Summarise chapter', 'Quiz me', 'Review flashcards']
  return ['Summarise chapter', 'Quiz me', 'Make flashcards']
}
```

### `src/components/FlashCard.tsx`

```typescript
interface FlashCardProps {
  front: string
  back: string
  isReview?: boolean   // shows Prev / Next / Mark as known buttons
}
```

Flip animation uses Framer Motion `animate={{ rotateY: flipped ? 180 : 0 }}` on the card container with `perspective` set via inline style. The front and back faces each use `style={{ backfaceVisibility: 'hidden' }}` (valid React inline style). This avoids CSS class requirement while working correctly in Safari.

Front face: warm gradient background (`linear-gradient(135deg, #F8F7F4, #F0EDE8)`), Lora italic 14px.
Back face: dark background (`#1A1917`), DM Sans 13px, `#F0EDE8` text.

### `src/components/Scratchpad.tsx`

```typescript

  isOpen: boolean
  onClose: () => void
  content: string
  onChange: (s: string) => void

Plain `<textarea>`, no rich text. DM Sans 14px, `lineHeight: 1.7`, `background: var(--bg-secondary)`, `border: none`, full height. Header shows "Scratchpad" label + saving indicator ("Saving…" / "Saved") + close button.

## `useEpub` Changes

Add `getFullChapterText(): string` to the `UseEpubReturn` interface and implementation:

```typescript
// In UseEpubReturn interface — add alongside getCurrentText:
getFullChapterText: () => string
```

Implementation: calls `rendition.getContents()` which returns the full rendered chapter document (not just the visible page), then extracts `contents.document.body.innerText`. Returns `''` if no chapter is loaded. This gives complete chapter text regardless of layout mode (scroll or paginated).

## Reader Changes

### Header layout

```
← ≡  Title  ·  3/142                [🔍][🔖][✎][📓][✦][☀]
```

New buttons (left to right in right section):
- `📓` (notebook icon) — toggles Scratchpad panel
- `✦` — toggles Study panel

The ✦ button uses `background: #1A1917` (filled) when the Study panel is open, `background: transparent` otherwise. The notebook button follows the same pattern.

### Right-side panel slot

Three panels share the right overlay slot (only one open at a time):
1. Notes pane (existing, toggled by ✎ Notes button)
2. Scratchpad (new, toggled by 📓)
3. Study panel (new, toggled by ✦)

Opening one closes the others.

### Chapter note floating button

A `+ Note` floating action button sits at the bottom-right of the epub container, `position: absolute`, `bottom: 60px` (above nav bar), `right: 16px`, `zIndex: 10`. Style: `background: var(--bg-surface)`, `border: 1px solid var(--border)`, `borderRadius: 20px`, `padding: 6px 14px`, DM Sans 12px. Tapping it opens a small inline note editor (same visual as SelectionBubble but without the quote preview). On save, dispatches an annotation with `type: 'chapter_note'` and `quote: ''`.

### Supabase write-through for chapter notes

`Reader.tsx` already calls `saveAnnotationToSupabase` after `addAnnotation` succeeds for highlight annotations. The same code path handles chapter notes — `addAnnotation` now receives `type: 'chapter_note'`, and the `useAnnotations` hook passes `annotation.type` to `saveAnnotationToSupabase` (which now forwards it rather than hardcoding `'note'`). No separate write path is required.

### Context assembly

`Reader.tsx` assembles `StudyContext` and re-memos it when dependencies change:

```typescript
const studyContext = useMemo<StudyContext>(() => ({
  chapterText: epub.getFullChapterText(),
  chapterNotes: annotations
    .filter(a => a.href.split('#')[0] === currentHref.split('#')[0] && a.type === 'chapter_note')
    .map(a => a.note),
  scratchpad: scratchpad.content,
  annotations: annotations
    .filter(a => a.type === 'note' && a.note)
    .map(a => ({ quote: a.quote, note: a.note })),
  flashcards: flashcards.map(f => ({ front: f.front, back: f.back })),
  selectedText: currentSelection ?? null,
}), [annotations, currentHref, scratchpad.content, flashcards, currentSelection])
```

### `onStudy` callback for BookDetailModal

`BookDetailModal` receives a new optional prop:

```typescript
onStudy?: (opts: { panel?: 'scratchpad'; chapterHref?: string }) => void
```

When `onStudy` is called, the parent (`Library.tsx` / `Landing.tsx`) opens the reader (`onRead()`) and passes the options via a short-lived ref or a `studyOptions` state variable. `Reader.tsx` reads this on mount and if `chapterHref` is set, opens the Study panel in review mode for that chapter. If `panel === 'scratchpad'`, opens the Scratchpad panel.

## Sidebar Changes

The Sidebar **Notes tab** renders a unified list. Each item is checked for `type`:

- `type === 'note'` → existing `AnnotationCard` with italic Lora quote block + note text
- `type === 'chapter_note'` → `NoteCard` variant: no quote block, just note text + "Chapter note" label + date

The `deleteAnnotation` handler in the sidebar works identically for both types (uses `id`).

The existing **export functions** in `exportAnnotations.ts` need updating:

**Markdown export** — branch on `type`:
```typescript
// type === 'note' (existing):
> ${a.quote}
${a.note}

// type === 'chapter_note' (new — no quote block):
📝 Chapter note
${a.note}
```

**JSON export** — pass `type` through in the exported object (it is already present on the `Annotation` interface after the migration).

## Book Detail Modal Changes

**New props:**

```typescript
onStudy?: (opts: { panel?: 'scratchpad'; chapterHref?: string }) => void
```

**On mount**, load in parallel:
1. `getAnnotationsFromSupabase(supabase, bookId)` (existing)
2. `getFlashcards(supabase, bookId)` (new)
3. `getScratchpad(supabase, bookId)` (new — returns full content; the modal renders only the first 200 characters as a preview, truncated in JSX: `scratchpad.slice(0, 200)`)
4. `loadProgress(supabase, bookId)` (existing)

**Stats card** replaces the standalone reading-time `<p>` in the left panel. Four rows: Reading time, Flashcards (gold if > 0), Chapter notes (gold if > 0), Highlights. The chapter notes count is `annotations.filter(a => a.type === 'chapter_note').length`.
